# SafeK Summer: 외국인 여행자를 위한 한국 안전 관광 가이드 v1.0 ☀️☔

## 1. 프로젝트 개요 (Product Requirements Document)

본 문서는 K-관광 중 극단적 기상 이변(폭염, 장마 등)에 노출된 외국인 관광객들에게 실시간 기상/안전 정보 및 내 주변 위치 기반 기능성 관광 코스와 편의시설 안내를 제공하는 **'SafeK Summer'**의 1차 MVP 구축 계획 및 산출물을 정의합니다.

### 1.1 기획 배경 및 비전

* **문제 정의:** 기상 이변(폭염 및 게릴라성 호우)과 낯선 지리/언어 환경으로 인한 외국인 관광객의 불안감 증가 및 관광 만족도 저하
* **해결책 (Solution):** 단순한 명소 나열이 아닌, '현재 날씨', '지하철 혼잡도' 및 '내 주변 실시간 상황'을 반영한 안전 대체 코스 필터링 및 화장실/안내소/대피소 등 편의시설 즉시 탐색 기능 제공.
* **기대 효과:** 서울 중심 MVP 런칭(1단계) 후, 전국 단위(2단계: 부산, 제주 등) 확장을 통한 K-관광 인프라 개선 효과.

## 2. 모노레포(Monorepo) 폴더 구조

심사위원 및 평가자가 프로젝트를 직관적으로 파악할 수 있도록 프론트엔드와 백엔드를 하나의 GitHub 레포지토리에서 관리하는 모노레포 구조를 채택하였습니다.

```plaintext
SafeK_Summer_Workspace/
├── docs/
│   └── PRD.md                 # SafeK Summer 기획 및 산출물 문서 (본 문서)
├── backend/                   # FastAPI, PostgreSQL, Prophet 데이터 모델 코드
├── frontend/                  # React Native (Expo) 기반 모바일 화면 코드
├── data_samples/              # 서비스 구동을 위한 수집 샘플 데이터 (Weather, Places, Subway 등)
├── docker-compose.yml         # 로컬 DB/캐시 실행을 위한 도커 설정
└── README.md                  # 프로젝트 요약, 실행 방법, 팀 소개
```

## 3. 로컬 기반 시연 (Demonstration) 가이드

평가자가 "이게 진짜 로컬에서 돌아가나요?"라고 물었을 때 즉각적인 시연이 가능하도록 준비된 1차 개발 완료 버전의 실행 가이드입니다.

### 3.1 백엔드 시연 준비

1. 터미널에서 `docker-compose up -d`를 실행하여 통합 DB(PostgreSQL)와 캐시 서버를 컨테이너로 구동합니다.
2. `backend/` 폴더 진입 후 FastAPI 서버를 실행합니다. (`uvicorn app.main:app --reload`)
3. 브라우저에서 `http://localhost:8000/docs` (Swagger UI)에 접속하여 **'API가 실제로 작동하며 추천 코스, 날씨, 혼잡도 데이터를 반환하는 화면'**을 띄워둡니다.

### 3.2 프론트엔드 시연 준비

1. 터미널에서 `npx expo start`를 실행하여 로컬 디바이스 또는 시뮬레이터에서 프론트엔드 앱을 구동합니다.
2. 가장 시각적 효과가 좋은 **'날씨 연동형 홈 화면'**과 '내 주변 화장실/안내소 맵 화면' 위주로 클릭하며 시연 시나리오를 진행합니다. (폭염 경고 배너 및 안심 대피소 팝업 등 시연)

## 4. 데이터 파이프라인 및 수집 샘플 증빙

실제 공공 API를 연결해 두었더라도, "우리가 이런 형태의 원천 데이터를 수집하고 정제/가공했다"는 점을 명확히 증빙하기 위해 `data_samples` 폴더에 다음과 같은 정적 파일을 구성해 두었습니다.

* `sample_weather_forecast.json` : 기상청 단기예보 API 등에서 받아온 격자별 날씨 및 폭염/호우 특보 원본 데이터 샘플.
* `sample_safe_places_cleaned.csv` : 행안부 공공화장실 데이터 및 관광공사 안내소 데이터를 DB에 직접 넣기 좋게 위경도, 영문명, 카테고리로 1차 정제(Cleaned)한 엑셀(CSV) 파일.
* `sample_subway_congestion.csv` : 서울교통공사의 지하철 시간대별 혼잡도 과거 데이터 샘플 (Prophet 시계열 모델의 혼잡도 예측을 위한 학습용 증빙 데이터).

## 5. 제품기획/제품소개서 (Pitch Deck) 구성안

심사위원이나 투자자를 효과적으로 설득하기 위해 핵심 내용만 찌르는 시각적 5장 구성의 Pitch Deck(PPT) 구조입니다.

* **Slide 1 (표지):** SafeK Summer - 외국인을 위한 실시간 날씨/안전 기반 스마트 관광 가이드
* **Slide 2 (문제 정의):** 기상 이변(폭염/장마)과 낯선 환경으로 인한 외국인 관광객의 불안감 증가 (관련 통계 및 뉴스 인용).
* **Slide 3 (해결책):** 단순 명소 추천이 아닌, '현재 날씨'와 '내 위치' 기반의 안전한 대체 코스 및 편의시설(화장실/안내소) 즉시 탐색 기능 제안.
* **Slide 4 (핵심 기술 및 화면):** 앞서 설계한 FastAPI 파이프라인 아키텍처 다이어그램과 React Native 앱 UI 스크린샷 2~3장 배치 (기술적 신뢰감 확보).
* **Slide 5 (기대 효과 및 비전):** 서울 중심 MVP 런칭 후, 전국 단위(부산, 제주 등) 확장을 통한 K-관광 인프라 전반의 체질 개선 효과.
